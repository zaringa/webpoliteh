# Лабораторная работа №8

## Тема
Форма регистрации и авторизации с JWT-валидацией, хранением состояния и навигацией между состояниями.

## Цель работы
Научиться создавать формы с JWT-валидацией, хранением состояния и навигацией между состояниями.

## Постановка задачи
Разработать страницу регистрации и авторизации для личного кабинета студента с:
- обработкой JWT-токенов;
- взаимодействием клиент-сервер;
- хранением пользователей в базе данных;
- защитой маршрутов;
- выходом из системы и очисткой токена.

## Реализация
- Использован `Tailwind CSS` для интерфейса (минимальный дизайн).
- Реализован backend на `Node.js + Express`:
  - `POST /api/auth/register` — регистрация;
  - `POST /api/auth/login` — вход;
  - `GET /api/me` — получение текущего пользователя (только с JWT);
  - `GET /api/protected/dashboard` — защищенный маршрут;
  - `POST /api/auth/logout` — выход и отзыв текущего токена.
- Использована база данных `SQLite` (`data/lab8.db`):
  - таблица `users`;
  - пароли хранятся в зашифрованном (хешированном) виде через `bcryptjs`.
- JWT-защита:
  - генерация токена при входе/регистрации;
  - middleware проверяет Bearer-токен и доступ к защищенным маршрутам.
- Состояние на клиенте:
  - переключение режимов `вход/регистрация`;
  - хранение JWT в `localStorage`;
  - восстановление сессии после перезагрузки;
  - выход из системы с очисткой токена.

## Файлы работы
- `server.js` — сервер, API, JWT middleware, работа с БД.
- `public/index.html` — интерфейс страницы.
- `public/script.js` — логика форм, состояние, работа с API.
- `package.json` — зависимости и скрипты запуска.

1. Форма авторизации и регистрации корректно обновляет состояние: выполнено.

3. Реализовано API для аутентификации и авторизации: выполнено.
4. Используется JWT для защиты маршрутов: выполнено.
5. Реализован выход из системы и очистка токена: выполнено.
6. Отчет оформлен согласно требованиям: выполнено (данный файл).
7. Отчет загружен в СДО: выполняется студентом.
8. Работа защищена очно преподавателю: выполняется студентом.
9. Обязательное условие для профиля дизайн (Tailwind CSS): выполнено.
