# Лабораторная работа №7

## Тема
Управление состоянием и загрузкой данных с помощью `useState` и `useEffect`, модульная архитектура и передача состояния между компонентами.

## Цель работы
Научиться управлять состоянием и загрузкой данных с помощью веб-хуков `useState` и `useEffect`, освоить принципы модульной архитектуры и передачи состояния между компонентами.

## Постановка задачи
Разработать страницу с игровым таймером с тремя счетчиками (время, очки, жизни), где:
- состояние сохраняется при перезапуске;
- эффекты выполняются при изменении значений (звук, смена фона, запись рекорда).

## Реализация
- Использованы `React` (хуки `useState`, `useEffect`) и `Tailwind CSS`.
- Реализована модульная архитектура компонентов:
  - `CounterCard` — карточка счетчика;
  - `ControlPanel` / `ControlButton` — управление игрой;
  - `App` — родительское состояние и эффекты.
- Локальное состояние (`useState`):
  - `timeLeft`, `points`, `lives`, `running`, `bestScore`, `bgMode`.
- Работа `useEffect`:
  - сохранение/восстановление состояния через `localStorage`;
  - интервал таймера и синхронизация счетчиков;
  - остановка игры при достижении нуля;
  - звуковые эффекты при критическом времени и потере жизни;
  - смена фона по состоянию (`normal`, `warning`, `danger`);
  - автоматическая запись нового рекорда при росте очков.
- Передача состояния между компонентами:
  - значения и обработчики передаются через `props` из `App` в дочерние компоненты.

## Файлы работы
- `index.html` — подключение Tailwind/React и корневой контейнер.
- `script.js` — компоненты, хуки состояния, эффекты, логика таймера.

3. Счетчики синхронизированы между собой: выполнено.
4. При достижении определенных значений срабатывают звуковые и визуальные эффекты: выполнено.
